# Glossary

| Term | Definition |
|------|-----------|
| **æ** | The invariant dimension. Pronounced "ash." Represents things that must not change. |
| **æ guard** | A node type that watches for drift from declared invariants. |
| **æ loom** | Midio's runtime engine. Weaves agent nodes together through yarn. |
| **Canvas** | The visual workspace where nodes are placed and connected. |
| **Drift** | When actual system behavior diverges from intended behavior. |
| **Molt** | A deliberate system evolution — re-evaluate, rewind, re-weave. Not a hotfix. |
| **Node** | A unit of agency on the canvas. Can be a source, transform, action, guard, or control. |
| **Swarm** | A collection of connected nodes working as a coherent system. |
| **Topology** | The spatial arrangement and connection pattern of nodes. |
| **Yarn** | Typed connections between nodes. Carries data, signals, state, or guard assertions. |
| **4D Manifold** | The conceptual space of Midio: space, time, agency, invariant. |
| **Ritual** | A scheduled workflow (e.g., the 8 AM morning ritual). |
